# AGENTS.md

## Project
DSC2204 ITP — Timetabling System for SIT Engineering Cluster.

## Role
You are an implementation assistant. ChatGPT is the project architect. Follow the existing architecture and avoid unnecessary redesign.

## Tech Stack
Python 3.11, pandas, openpyxl, pytest.

## Architecture
Input Excel/CSV -> loader -> dataclasses -> constraint checker -> greedy scheduler -> local search optimiser -> Excel exporter.

## Coding Rules
- Use existing dataclasses in `data/models.py`.
- Use `config.py` for constants.
- Keep functions small and single-purpose.
- Add type hints.
- Add short docstrings.
- Avoid new dependencies beyond pandas, openpyxl, pytest, and standard library.
- Preserve clear separation between hard and soft constraints.

## Scheduling Rule
Hard constraints define feasibility. Soft constraints define quality. Optimisation must never introduce hard violations.

## Required Hard Constraints
- No two classes in the same room at the same time.
- A tutor cannot teach two classes at the same time.
- A student group cannot attend two classes at the same time.
- Room capacity must be >= class enrolment.
- ONLINE classes must use virtual room only.
- F2F classes must not use virtual room.
- Odd/even week pattern must be respected.
- No classes on public holidays or term breaks.
- No classes before 09:00 or after 18:00.
- No Wednesday afternoon classes from 13:00.
- No Friday 12:00-14:00 classes.
- No Saturday classes.
- No Friday classes after 17:00.
- Lunch must provide each group at least 1 free hour between 11:00 and 14:00.

## Required Soft Constraints
- Avoid Online/F2F switch in adjacent slots for same tutor.
- Avoid Online/F2F switch in adjacent slots for same group.
- Avoid tutor idle gaps longer than 2 hours.
- Avoid student group having more than 4 consecutive teaching hours.
- Avoid more than 3 consecutive back-to-back group hours where possible.
- Cluster tutor classes to minimise wasted free slots.
- Prefer room utilisation >= 60%.
- Avoid first/last slot for groups.
- Prefer classes to end by 17:00.

## Common Modules
Common modules should be scheduled once for all linked cohorts, using combined enrolment and a room large enough to host everyone. Preserve linked cohort/group identifiers so group clash and lunch checks remain valid.

## Output Rule
`Room1` in `final_timetable.xlsx` must contain the assigned room ID generated by the algorithm:

```python
"Room1": assignment.room.room_id if assignment.room else None
```

Use the same assigned room ID for `Location Hostkey` where possible.

## Validation
Run:

```bash
python main.py
pytest
```

Success means:
- Hard violations = 0.
- Output Excel files generated.
- `Room1` populated with assigned room IDs.
- Tests pass.
- Soft violations are reported and preferably reduced after optimisation.
